1. Design — Why Context-Only

The v1 lesson

v1 used 68 dimensions: 24-dim category one-hot + 5-dim tier one-hot + 37 context dims. Category kNN accuracy was 99.5% — but this was trivial: the one-hot dominated, and removing identity caused a −45.9% drop while removing all context layers caused ~0% drop. The vector WAS the category label, decorated with noise.

The v2 fix

Category and tier are metadata, not features. The 37-dim vector encodes only spatial context, hex context, influence, and competitive position. Two places with similar vectors serve the same kind of area regardless of what category they are. A premium cafe in Orchard and a premium restaurant in Orchard should be NEAR each other — they share the same urban context.

LayerDimsWhat it encodes
Spatial context (micrograph)12T1-T4 weights, anchor count, competitive pressure, density band
Hex context14Population density, land use, commercial density, walkability, MRT, tourist draw, dev gap
Influence7Spatial + transit max-influence for same category, walking neighborhood avg
Competitive position4Tier rank in hex, brand rarity, category share, category rank

2.3 Category prediction from context alone

Metricv1 (with one-hot)v2 (context-only)What it means
kNN k=1 category accuracy99.5%56.7%Context alone predicts category 57% of the time — real but not dominant
kNN k=5 category hit rate99.8%80.9%Within top-5 neighbors, 81% share the category

56.7% is above the 4.2% random baseline (1/24 categories) by 13.5x. Context IS predictive of category — hawkers cluster near HDB, bars cluster near hotels, cafes cluster near MRT. But it’s no longer trivially dominant. The 43.3% "errors" are the valuable cross-category matches: a cafe's nearest neighbor is a restaurant in the same mall, because they share the same urban role.

4. What Place2Vec v2 Enables

Use caseHow Place2Vec helps
Site selection"Where should Starbucks open next?" → Find hexes where existing place vectors match the Starbucks-in-successful-CBD context (high transit T1, moderate competition, premium walkability)
Refined gap analysis"This hex has cafes but they're all budget-tier in a premium-context area" → context vector shows the area demands premium, but actual places are mismatched
Co-location recommendations"What other categories thrive in contexts like this?" → Find all top-performing places with similar context vectors, regardless of category
Competition mapping"Who are my REAL competitors?" → Not just same-category, but same-context places that fight for the same customer in the same walkability zone
Region poolingMean-pool place vectors per hex → hex "commercial DNA" embedding that captures what KIND of places, not just how many (0.332 PA accuracy vs 0.130 for v1)
