1. What This Model Does

For each hex, mask some place categories (e.g., hide the restaurant count, the cafe count) and predict them from:

  • Physical context (391 features): buildings, population, transit, walkability, land use, micrograph, influence graph features — everything EXCEPT place counts
  • Visible categories: the unmasked category counts (e.g., hawker stalls and education are visible, restaurants and cafes are masked)
  • Graph neighborhood: what the hex's spatial and transit neighbors look like (2-hop GCN propagation)

The model learns: "given this infrastructure, these visible categories, and what the neighborhood looks like — what ELSE should exist here?"

This is BERT for urban space. The masked categories are the [MASK] tokens. The graph provides the sentence-level context.

5. Why the GCN Beats Everything Else

The graph adds real structure that features cannot capture

Our 460-feature table includes 123 influence features (sp_max_*, tr_max_*, etc.) that encode "what's reachable." But these are pre-computed aggregates — they capture the max and mean of the neighborhood, not the full neighbor distribution.

The GCN sees the actual neighbor features and propagates information through 2 hops. It knows not just "the densest nearby hex has 1,356 places" but "the path from here through 2 intermediate hexes reaches a specific commercial pattern." This is richer than any pre-computed feature.

The dual graph captures complementary signals

Spatial GCN (48K edges): what's within walking distance. Propagates local context — "my neighbors are HDB, so I'm in a heartland."

Transit GCN (13K edges): what's reachable by MRT. Propagates long-range context — "Bedok is connected to CBD via the East-West Line, so Bedok knows about CBD commercial density."

The learned α weight balances both. The model discovers the optimal mix of local and long-range context for predicting each hex's commercial composition.

Masked prediction is a better training signal than reconstruction

Autoencoders optimise: "compress and decompress 460 features." The bottleneck allocates capacity to high-variance features (noise).

Masked prediction optimises: "given infrastructure, predict commerce." The model MUST learn the infrastructure→commerce relationship. The embedding encodes urban character because that's what predicts masked categories.
