1. What We Built

A 305-dimensional feature representation for each of 7,318 H3 resolution-9 hexagons (~175m edge) covering Singapore. The representation unifies 14 authoritative data sources into a single row-per-hex table that can be consumed by any downstream model — autoencoder, GNN, regression, clustering.

2. Key Experiment: Spatial + Transit Influence Graph

The core finding: what commercial center a hex can reach (spatially and by transit) is dramatically more informative than naive k-ring averaging.

2.1 Aggregation Method Comparison (k=5, 30 features each)

Metric: kNN k=5, planning-area match accuracy on 3,934 active hexes.

MethodAccuracyLift
Self + lat/lng (baseline)0.242—

+ Transit only (max + pw)0.444+0.202
+ Spatial + Transit combined0.587+0.345

Finding: Max-influence dominates averaging methods

The single densest commercial hex within walking reach (+0.132) beats uniform mean (+0.080), gravity (+0.072), and pop-weighted (+0.069). The representation cares about what's the biggest thing near me, not what's the average around me.

Finding: Spatial + transit are complementary, not redundant

Spatial alone: 0.438. Transit alone: 0.444. Combined: 0.587. The 0.587 is much more than either alone — they capture different neighborhoods. A Bedok HDB hex sees residential spatial neighbors but CBD transit neighbors. That dual view is the key signal. Correlation between spatial_max and transit_max = 0.605 (moderate, complementary).

3. What We Discovered (and Fixed) Along the Way

3.1 Broadcast Leakage (40 features were fake)

The original V9 hex table had 40 features with zero within-subzone variance — they were copied identically to every hex in a subzone (or planning area). These made the representation look richer than it was:

Feature familyColsRoot causeFix
NVIDIA personas35Source file has only 48 unique rows across 318 subzones — data is actually at planning area level, not subzoneDropped at hex level entirely
HDB resale prices2Subzone median broadcast to all hexesDropped (no geocoding for per-hex)
elderly_pct, walking_dep_pct2Ratio of two dasymetric quantities = subzone constant (numerator/denominator scale identically)Keep counts, drop ratios
avg_gpr1Subzone average broadcastRecomputed from URA parcel intersection (now has real within-subzone std 0.34)

After fix: 0 broadcast columns.

3.2 Missing Subzones (VivoCity wasn't in the map)

The V9 hex universe was missing 11 subzones including Maritime Square (VivoCity/HarbourFront — Singapore's largest mall), Sentosa (Universal Studios, resort hotels), and Siglap (mainland HDB). 8,131 places had no hex home.

After fix: 7,318 hexes (up from 5,897), 327/332 subzones covered. Two-pass polyfill (center containment + overlap mode) with majority-area subzone assignment for boundary hexes.

3.3 HDB Block Count Mismatch

The fused building table marked only 10,431 buildings as is_hdb=True vs 13,386 in the authoritative HDB geojson. Fixed by direct spatial join from the authoritative source. Now exact: 13,386.

3.4 Contrast and Rank Features Hurt Representation

The original k-ring approach computed 5 aggregates per neighbor feature: mean, max, contrast (self minus mean), rank (percentile within neighbors). Testing revealed contrast (-1.2%) and rank (-2.9%) actively degrade nearest-neighbor quality. They encode local structural position (useful for "am I a peak?") but not geographic context (useless for "am I similar to that hex across town?").

8. Honest Assessment: Are We on the Right Track?

YES — the foundation is solid

  • Data integration is comprehensive. 14 sources, 174K places, 4.2M residents, 377K buildings, 113K zoning parcels, 231 MRT stations — all properly joined to 7,318 hexes with verified totals.
  • The influence graph is a real finding. +34.5% lift beyond position is genuine contextual signal. The spatial+transit dual-view is the key architectural idea — not just encoding "where am I" but "what can I reach."
  • Feature quality is validated. 32/33 checks pass. No broadcast. All landmarks spot-check correctly. Archetypes emerge unsupervised. Structural twins (Sentosa ↔ MBS) are found across PA boundaries.
  • The pipeline is reproducible. 13 scripts, deterministic, each reads from source and writes a parquet. The full chain rebuilds in ~2 minutes.

NOT YET — what's missing before this is a "universal representation"

  • No learned embedding. We have a 305-feature row, not a compressed embedding. An autoencoder (305 → 64 → 32) or GNN over the influence graph would learn what matters. That's the next step.
  • No downstream task evaluation. kNN PA-match is a proxy metric. We need to show the representation actually improves real predictions: HDB resale price, place count forecasting, gap analysis. The data is all here — just need to plug it in.
  • Single city. "Universal" requires showing the same framework works on at least one more city. We have NYC data at /Users/sumanth/propheus-projs/da-sgp/digital-atlas-v2/.
  • Micrographs are weak. v2 is 66K places with duplicated per-category files. v3 (174K, per-category) is on the server but not synced. The 20 micrograph features are the least validated pillar.
  • No comparison to existing methods. We haven't benchmarked against Region2Vec, Urban2Vec, or even simple PCA on census data.
  • The influence graph is heuristic. k=5, 5km station threshold — hand-tuned. A graph attention network that learns which neighbors matter would be the paper-ready version.
