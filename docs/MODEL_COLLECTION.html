2. The Four Representations

A. Raw Feature Table (460 features)

Hand-engineered from 14 data sources: buildings, population, transit, walkability, place composition, micrograph v3, influence graph, development gaps, satellite-inspired features, mobility. Zero NaN, zero broadcast, 120/120 ground-truth verified.

StrengthsBest for supervised prediction (R²=0.896 for category counts). Complete, interpretable, validated.
Weaknesses460 dims is wide. Hard to share. Cosine similarity is suboptimal (kNN=0.374).
Best forTraining downstream models (XGBoost, Ridge, neural nets) when you have labels.

B. GCN-64 Embedding (64 dims)

Learned by a dual-graph GCN (spatial + transit) with masked category prediction. The ONLY method that beats raw features on kNN similarity.

StrengthsBest similarity (kNN=0.430). Captures graph structure. Gap analysis as decoder product. Compact (64 dims).
WeaknessesLoses supervised prediction power (R²=0.649 vs 0.896). Needs PyTorch to re-train. Not interpretable per dimension.
Best forSimilarity search, gap analysis, sharing, plugging into downstream models.

C. PCA-128 (128 dims)

Linear projection of the 460-feature table. 94.5% variance preserved. Each component is a named weighted combination of original features.

StrengthsInstant (no training). Deterministic. Interpretable components. 0.359 kNN. Re-usable PCA model for new hexes.
WeaknessesLinear only — misses non-linear patterns. Doesn’t use graph structure. kNN weaker than GCN.
Best forQuick compression when you don’t need a neural network. Visualization (UMAP on PCA-128).

D. Place2Vec v2 (37 dims per place)

Context-only place vectors for 169K individual places. No category one-hot — the vector encodes WHERE a place is, not WHAT it is.

StrengthsPer-place granularity. Cross-category neighbors (McDonald’s finds Starbucks in same mall). Hawker archetype analysis.
Weaknesses57% of vector is hex-level features (same for all places in a hex). Not a learned embedding. Limited per-place differentiation.
Best forPer-place context lookup. Competitive analysis. Hawker/food analysis. NOT for place-to-place similarity (use with caution).

The key insight: only graph-based methods (GCN) break through the feature ceiling. Standard autoencoders, masked autoencoders, and contrastive methods all LOSE to PCA — because 7K hexes is too small for non-linear methods to outperform linear compression. The graph adds genuine structural information that no amount of feature engineering captures.
