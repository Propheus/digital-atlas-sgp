3. Why Each Method Wins Where It Wins

Node2Vec-64 (80% overall, 4 category wins)

Why it dominates: random walks on the 47K-edge influence graph + skip-gram directly optimize for "graph-neighbor similarity." Many tests (pa_coherence, landmark, cross_pa_transfer, robustness) reward exactly that. SGP's influence graph is PA-clustered (MRT+roads honor admin boundaries), so graph-structural similarity naturally aligns with PA membership.

Weakness: no feature awareness. On feature_query (hexes with extreme feature values should cluster) it scores 50% vs GCN's 100%.

GCN-64 (72% overall, 2 unique wins)

Why it wins two categories outright:

  • • archetype_stability 100% vs Node2Vec 60%: k-means on GCN gives stable clusters across random seeds. Node2Vec does not.
  • • feature_query 100% vs Node2Vec 50%: GCN can answer "find hexes with profiles like this profile" because it knows features.

Weakness: loses to Node2Vec on PA coherence / landmarks because its training objective is masked category prediction, not similarity. Graph information reaches it only through features, not directly.

UMAP-64 (71% overall, 1 category win)

Why it wins functional: nonlinear manifold learning preserves local feature-space structure. Hexes with similar feature profiles (e.g. all Tampines HDB) land close together in 64-d. Competitive on feature_query (100%).

Weakness: no graph. Weak on pure graph-structural tests (robustness 20%).

Transformer-64 (59% overall, 0 wins)

Why it underperforms: masked-feature reconstruction with 32-d self-attention doesn't produce similarity-aligned embeddings. Reconstruction loss optimizes for feature-completion accuracy, not embedding separation. Competitive only on feature_query (100%) where the CLS token happens to cluster feature-extreme hexes.

Lesson: architecture ≠ performance. Transformers are powerful for sequences; for tabular feature bags, there's no obvious gain over simpler methods.

XGBoost-PCA-64 (43% overall, 0 wins)

Why it's weakest: XGBoost leaves compress variation that matters for predicting pc_cat_* counts. Similarity is orthogonal. This confirms the earlier finding: XGBoost excels at prediction (R²=0.80), not representation.

5. The Headline Insight

GCN-64 is the only embedding in this comparison that combines graph structure AND feature awareness in a single vector.

Node2Vec has graph only. UMAP has features only. Transformer has features only. XGBoost has (supervised) features only. GCN dominates the two categories (archetype_stability, feature_query) that require both axes of signal.

For downstream use cases where clustering stability or feature-profile retrieval matters, GCN-64 is irreplaceable — Node2Vec wins overall pass rate but cannot do these tasks.

The production argument is not "ship one embedding, ship the best." It's "ship three complementary embeddings": Node2Vec for graph-structural similarity, GCN-64 for feature+graph hybrid tasks, UMAP for nonlinear manifold retrieval. Documented per use case. Combined bundle size: ~8 MB.

6. Methodology

Data: 7,318 H3 res-9 hexes covering SGP. 391 raw features (normalized). Influence graph with 47K edges (spatial + transit).

Embedding configs:

  • • GCN-64: dual-graph (spatial + transit) + context encoder + target encoder with masked prediction. Training still ongoing — best config found so far via continuous hyperparameter search (engine at cycle 65+).
  • • Node2Vec-64: 10 walks × 80 steps per node, window=10, skip-gram, 5 epochs.
  • • UMAP-64: n_neighbors=30, min_dist=0.1, metric=cosine.
  • • Transformer-64: each feature as token, 2-layer × 2-head × 32-d, masked-feature reconstruction 15% mask, 40 epochs.
  • • XGBoost-PCA-64: 24 XGBoost predictors → leaf-index matrix → PCA to 64d.

Test suite: 260 tests across 10 categories: PA coherence (50), landmark retrieval (30), anti-similarity pairs (30), graph smoothness (30), archetype stability (20), functional equivalence (10), feature query (20), triplet ranking (30), cross-PA transfer (20), robustness (20).

Scoring: each embedding z-standardized per dim, then L2-normalized for cosine. Pass rate = fraction of tests whose criterion is satisfied (e.g. "top-5 neighbors share archetype family").
